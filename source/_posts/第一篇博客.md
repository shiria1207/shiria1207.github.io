## blog搭建

第一篇blog就来记录搭建blog踩的坑吧
我日常使用的配置是`win11 + vmware + ubuntu22.04`，由于用不惯`powershell`但是平时比赛`writeup`都是在物理机写的, 而且虚拟机开机太慢了, 于是这次顺便配置了`wsl2 + kali-linux`.

本次搭建blog翻了大车, 两次删库重来, 第一次随便找了个教程最后的连接远端仓库使用了`$ hexo d`命令, 但这样本地就没有git仓库, 第二次下载butterfly是直接git clone,但是没处理好导致.git在子目录下从而使两个仓库冲突.
第三次重新找了[Yime师傅的教程](https://blog.r1ce.cn/eb5fa6bf3c86/)

前面没啥问题, 特别注意如果删库重装还要重新安装butterfly依赖包`$ npm install hexo-renderer-pug hexo-renderer-stylus --save`, 这里没有加-g全局配置,所以删库时一起删了.

但是直接跟着一步步做到7.2push到远端仓库还是会有问题

```
❌Build with Jekyll
...
Error:  Logging at level: debug Configuration file: /github/workspace/./_config.yml Theme: butterfly github-pages 232 | Error: The butterfly theme could not be found. 
```

这说明github使用了Jekyll解析和搭建page, 但是我使用的是hexo
问ai和查阅[hexo官方文档](https://hexo.io/docs/github-pages)可以得知,需要先将仓库--settings--Pages--Build and development 的develop from a branch 改成 Github Actions, 然后在本地仓库添加 .github/workflows/pages.yml

```yml
name: Pages

on:
  push:
    branches:
      - main # default branch

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          token: ${{ secrets.GITHUB_TOKEN }}
          # If your repository depends on submodule, please see: https://github.com/actions/checkout
          submodules: recursive
      - name: Use Node.js 20
        uses: actions/setup-node@v4
        with:
          # Examples: 20, 18.19, >=16.20.2, lts/Iron, lts/Hydrogen, *, latest, current, node
          # Ref: https://github.com/actions/setup-node#supported-version-syntax
          node-version: "20"
      - name: Cache NPM dependencies
        uses: actions/cache@v4
        with:
          path: node_modules
          key: ${{ runner.OS }}-npm-cache
          restore-keys: |
            ${{ runner.OS }}-npm-cache
      - name: Install Dependencies
        run: npm install
      - name: Build
        run: npm run build
      - name: Upload Pages artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public
  deploy:
    needs: build
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

如果有买别的域名还需要额外修改, 但我没有, 跳过
成功push到远端仓库, 搭建阶段顺利结束, 开始研究主题配置

## butterfly一图流配置

> 本人使用的hexo和butterfly版本为Hexo 8.1.2 && Butterfly 5.5.4，一切步骤可行性基于这两个版本号，如果版本号不对可能无法复现，事实上本人跟着网上的教程一步步做也确实遇到了一些问题

一图流是butterfly不得不品的一环，搭建过程参考[Gehina师傅的教程](https://blog.ac2g.com/post/2624)
在blog根目录下，根据官方文档将./themes/butterfly/_config.yml复制到./_config.butterfly.yml，后续所有修改改在这里就行，生成web的时候hexo框架会自动合并这个config和themes里面的config
### 主题配置
_config.butterfly.yml其他部分可以自行研究，这里给出一图流部分必要的几块，**重点关注background url部分**

```yml
# The banner image of index page
# 这是刚进入主页时候的图片，往下划到文章部分的才是背景
# 设置为透明
index_img: transparent

# The background image of footer
# 这是最底下包含copyright，author等东西的部分背景
footer_img: transparent

# Website Background
# Can set it to color, image URL or an array containing colors and/or image URLs
# If an array is provided, a random background will be selected from the array on each load

# 这里设定背景图片链接，由于架构和index img，icon img等不一样，不能放诸如/img/xxx.png的相对链接，需要设置浏览器地址栏可以直接访问的静态链接

# 笔者是直接把图片扔进github仓库的，仓库里图片的静态链接需要点开图片后url后面加上 ?raw=true ，就会自动跳转到raw.github....的图片链接

# 大概率是因为版本问题，butterfly官方文档、网上大部分的教程、询问ai，都会告诉你这里应该写
# background: url(https://an_example.com/an_example_name.png)
# 但是这么写是加载不出来的，经浏览器f12查看前端代码后可以发现里面生成的图片链接变成了url(url(https://an_example.com/an_example_name.png))，去除url()后成功加载图片
background: https://an_example.com/an_example_name.png

# Add a mask to the header and footer
# 这里为true会给header或footer加上一层黑色薄纱的效果，看个人喜好设置
mask:
  header: false
  footer: false
```

### 安装插件脚本
先检查npm有无cheerio模块，没有则补装
```bash
$ npm install cheerio
```
在blog根目录新建文件夹scripts，和一个文件，位于 scripts/modify.js，并增加以下内容
```js
'use strict';
const { filter } = hexo.extend;
const cheerio = require('cheerio');

/**
 * 在页面插入新顶部图
 * @param {cheerio.Root} $ Root
 */
function insertTopImg($) {
    const header = $('#page-header');
    if (header.length === 0) return;
    const background = header.css('background-image');
    if (!background) return;
    $('#post, #page, #archive, #tag, #category').prepend(`<div class="top-img" style="background-image: ${background};"></div>`);
}

// 修改 HTML
filter.register('after_render:html', (str, data) => {
    const $ = cheerio.load(str, {
        decodeEntities: false
    });
    insertTopImg($);
    return $.html();
});
```
此处就已经配置好一图流了

### 配置文章界面的透明度
新建一个文件，位于 source/css/transpancy.css，并增加以下内容
```css
/* 文章页背景 */
.layout_post>#post {
    /* 以下代表透明度为0.7 可以自行修改*/
    background: rgba(255,255,255,.8);
}
 
/* 所有页面背景 */
#aside_content .card-widget, #recent-posts>.recent-post-item, .layout_page>div:first-child:not(.recent-posts), .layout_post>#page, .layout_post>#post, .read-mode .layout_post>#post{
    /* 以下代表透明度为0.7 */
    background: rgba(255,255,255,.8);
}
/*侧边卡片的透明度 */
:root {
  --card-bg: rgba(255, 255, 255, .8);
}
/* 页脚透明 */
#footer {
	/* 以下代表透明度为0.7 */
	background: rgba(255,255,255, .0);
}
```
根据需要调整.8来改变透明度
修改 Butterfly 的配置文件 _config.butterfly.yml，在 inject.head 选项引入样式
```yml
# Inject
# Insert the code to head (before '</head>' tag) and the bottom (before '</body>' tag)
# 插入代码到头部 </head> 之前 和 底部 </body> 之前
inject:
  head:
  # - <link rel="stylesheet" href="/xxx.css">
    - <link rel="stylesheet" href="/css/modify.css">
    # 在前一步配置的基础上增加
    - <link rel="stylesheet" href="/css/transpancy.css">
```
使用插件的方式实现，更新主题时，只要结构变化不大，无需任何操作即可继续使用
